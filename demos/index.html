Alpaca Demonstrations

This page provides a number of pre-built demonstrations of fully-constructed forms using Alpaca with a combination of effective technologies including jQuery, Twitter Bootstrap, jQuery Mobile and the Dojo Toolkit.

Furthermore, these demonstrations are packaged along with the Alpaca release so that developers can get a quick start on building their own forms. In many cases, existing forms can simply be taken and modified.

Developers

These demonstrations are located under the /demos directory that is shipped as a distribution. You can also download the source from GitHub.

If you choose to download a distribution, everything is already set. Just go to the /demos directory and all of the files you need will be there. Each demo is located in its own directory.

If you've synced the code from GitHub, then please do the following:

  1. Run ant to build Alpaca locally.
  2. Navigate to the /demos directory and you will see a list of demos.
  3. Within each demo folder, you can run ant to pull in local copies of all dependencies, such as Twitter Bootstrap, jQuery and, of course, Alpaca itself.

All of the dependencies will be pulled into a lib directory. Once done, your Alpaca form project will be entirely standalone. You can copy the folder to anyplace you'd like such as a PHP server, a NodeJS server or even right to your desktop.

Browser Demonstrations

Alpaca runs right inside a browser running on the web or on a mobile device. These projects introduce Alpaca to a web site by simply including the alpaca.css and alpaca.js files, respectively.

  • Quick Start
    An empty Alpaca project that you can use as a template for building your own forms ready-to-run in the browser. All of the Alpaca dependencies are loaded using SCRIPT tags. Adjust the source code to fit your own schema, options and layout needs.
  • Registration
    A sample registration form that uses a custom layout and custom validation messages. The results of the form are posted to a server-side PHP file for capture into a database. The storage into the DB is left as an exercise for the end user.
