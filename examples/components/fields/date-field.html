Date Field

Example #1: Simple date field

This is the simplest way that you can render a date. It's driven purely off of JSON schema and uses the format setting to specify that the value should be a date. Alpaca renders a date field by default.

Example #2: Invalid date field

Date fields participate in the validation chain along with everything else. Here is an example of a date field that is invalid. The hideInitValidationError setting is provided but it isn't necessary since it defaults to false anyway for editable views. You can set it to true to hide the initial validation message.

Example #3: Display date

Here we use the VIEW_WEB_DISPLAY view to tell Alpaca to render a set of fields in a display-only mode. Nothing is editable, including the date.
