Getting Started

What is Alpaca?

Alpaca is a client-side forms and templating engine which gives your application the ability to render elegant content entry and presentation forms with minimal development effort.

Start by checking out hosted demos

To get a feel for what Alpaca is capable of, check out the hosted demos. In the demos section, the left-side navigation lists sample forms (under Form Gallery) and all of the form controls and features (under Component Gallery) that Alpaca offers. Choose a sample form, control or feature and you'll be presented with several examples for that particular component. Each demo allows you to view source code by clicking on "Source Code" button.

Download Alpaca

Alpaca core library download contains both Alpaca JavaScript library and its supporting stylesheet. It is available in both minified format (for production) and uncompressed format (for development).

Download for Alpaca examples is a self-contained zip or war file which includes everything you need to run Alpaca examples locally.

Intro to using Alpaca

Once you've downloaded Alpaca core library, you'll get a zip containing the following

  • /js/
  • /css/

Unzip the zip file and then use your favorite text editor to create an empty html file. You will first need to include jQuery library (1.5 or later) and jQuery UI library (1.8.8 or later). jQuery UI library is not required for running Alpaca but highly recommended. 

                <script type="text/javascript" src="http://ajax.googleapis.com/ajax/libs/jquery/1.5/jquery.min.js">
                </script>
                <script type="text/javascript" src="http://ajax.googleapis.com/ajax/libs/jqueryui/1.8.8/jquery-ui.min.js">
                </script>
                <link type="text/css" href="http://ajax.googleapis.com/ajax/libs/jqueryui/1.8.8/themes/base/jquery.ui.all.css"
                rel="stylesheet">

Now include link to minified Alpaca JavaScript library and minified Alpaca stylesheet in its head section 

                <script type="text/javascript" src="js/alpaca-${project.version}.min.js">
                </script>
                <link type="text/css" href="css/alpaca-${project.version}.min.css" rel="stylesheet"/>

Or you can include uncompressed version

                <script type="text/javascript" src="js/alpaca-${project.version}.js">
                </script>
                <link type="text/css" href="css/alpaca-${project.version}.css" rel="stylesheet"/>

If you prefer to use hosted Alpaca library and stylesheet, all you need is to include 

                <script type="text/javascript" src="http://code.cloudcms.com/alpaca/${project.version}/js/alpaca-${project.version}.min.js">
                </script>
                <link type="text/css" href="http://code.cloudcms.com/alpaca/${project.version}/css/alpaca-${project.version}.min.css"
                rel="stylesheet"/>

Now add a simple test to the body section of your test html 

                <div id="field1">
                </div>
                <script type="text/javascript">
                $(function() {
                    $("#field1").alpaca({
                        "data": "Alpaca rocks!"
                    });
                });
                </script>

Save the html file and open it in a browser. You should get a simple text field with "Alpaca rocks!" in it.

To learn more about Alpaca library, you can also download Alpaca samples and have them running on your machine. Since Alpaca samples contain many examples that use Ajax calls, it is highly recommended to run the examples either in a web server (such as Apache Web Server ) or an web application server (such as Tomcat or Jetty).

For web application servers, you can simply download and deploy Alpaca war and then open your browser and point to index.html page of the deployed web application. The war file contains a proxy servelet for the Cloud CMS Connector example. Therefore no additional step will be required.

For web servers, you need to download and unzip Alpaca samples zip and then open your browser and point to the index.html page which is under the root directory of the unzipped file. In order to run the Cloud CMS Connector example, you will need to setup a reverse proxy that will forward request "/proxy/*" to "http://demo.cloudcms.net:80".

Here is a sample configuration for setting Apache proxy 

        ProxyRequests Off

        ProxyPassInterpolateEnv On
        ProxyPass /proxy http://api.cloudcms.com:80 interpolate
        ProxyPassReverse /proxy http://api.cloudcms.com:80 interpolate
        ProxyPassReverseCookieDomain localhost ${HTTP_HOST} interpolate

Build Alpaca from source

All Alpaca source code is kept under Git revision control, which you can browse online.

If you have access to Git, you can connect to the repository here: 

git@github.com:gitana/alpaca.git

Once you clone the Alpaca repository to your local machine and have Maven 3.0 or later installed, You can build Alpaca and run Alpaca examples using following Maven command 

mvn clean install jetty:run-exploded

If you want to run Cloud CMS Connector example, you will need to use following Maven command instead 

mvn clean install jetty:run-exploded -Pcloudcms

Wait till Maven finishes the build process and brings up embedded Jetty server, you should now be able to check out all Alpaca examples from this URL 

http://localhost:8180/alpaca/index.html
